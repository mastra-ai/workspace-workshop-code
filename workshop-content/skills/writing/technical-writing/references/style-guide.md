# Technical Writing Style Guide

## Headings

- Use sentence case: "Getting started" not "Getting Started"
- Keep headings concise (under 8 words)
- Use heading levels hierarchically (don't skip from h2 to h4)

## Lists

Use bullet points for:
- Unordered items
- Features or capabilities
- Requirements without sequence

Use numbered lists for:
1. Step-by-step instructions
2. Prioritized items
3. Sequential processes

## Tables

Use tables for:
- Comparing options
- Parameter/option documentation
- Reference data

| Column | Description |
|--------|-------------|
| Name | What it's called |
| Type | Data type |
| Default | Default value |

## Code Formatting

- Use `inline code` for:
  - File names: `config.json`
  - Function names: `createAgent()`
  - Variable names: `userId`
  - Command line input: `npm install`

- Use code blocks for:
  - Multi-line code
  - Command sequences
  - Configuration examples

## Word Choices

| Avoid | Use Instead |
|-------|-------------|
| utilize | use |
| in order to | to |
| a number of | several / many |
| at this point in time | now |
| in the event that | if |
| prior to | before |

## Punctuation

- Use Oxford commas: "agents, tools, and workflows"
- One space after periods
- No periods in headings
